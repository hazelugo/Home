**No-JS check (QUAL-03):**
1. Open index.html in a browser with JavaScript disabled (or inspect the HTML directly).
2. Confirm core content is present in the static HTML — hero, about paragraphs, skills table, workflow section, all projects (collapsed accordions are still readable because the project name + summary + tag are in the static row), experience timeline, contact links.
3. Confirm the sidebar nav anchors still work as plain `<a href="#section">` links (they do — they're standard anchors; only the active-state highlighting requires JS).

If this is judged insufficient, add a `<noscript>` block with CSS that sets `.project-expand { grid-template-rows: 1fr; }` to show all details when JS is disabled. Only add if needed.

**Impeccable-style check (QUAL-02) — verify against the design system in CLAUDE.md:**
- All new content (workflow section, AI projects, contact resume row, footer link) uses existing CSS tokens (var(--text), var(--accent), etc.) — no hardcoded colors
- All new interactive elements have accessible labels (links have text, the new contact-link has aria via the existing pattern)
- The new resume.html document SVG has `aria-hidden="true"`
- No new animations, no glassmorphism, no glow effects introduced
- Light theme preserved
- resume.html uses semantic HTML (h1, h2, header, section)
- resume.html link colors meet contrast requirements (#555 on white = ~AA)

Document any findings. If a Critical or High finding is identified, fix it in this task (small edits only) or escalate via a follow-up plan if substantial.
